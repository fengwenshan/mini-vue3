# Ref 响应式实现说明

本文档说明 `packages/reactivity/src/ref.ts` 中 `ref` 的实现方式，以及对应单元测试的设计思路。

## 一、ref 解决什么问题

`reactive` 只能代理对象：

```ts
const state = reactive({
  count: 1
})
```

但是基础类型无法被 `Proxy` 直接代理，例如：

```ts
const count = 1
```

`ref` 的作用就是把任意值包装成一个对象，通过 `.value` 来读写这个值：

```ts
const count = ref(1)

console.log(count.value) // 1

count.value = 2

console.log(count.value) // 2
```

这样即使是 number、string、boolean 这类基础类型，也可以拥有统一的响应式容器。

## 三、对外 API

当前导出了三个 API：

```ts
ref()
isRef()
unRef()
```

### ref

创建一个 ref 对象：

```ts
const count = ref(1)

count.value // 1
```

类型定义：

```ts
export interface Ref<T = any> {
  value: T
}
```

### isRef

判断一个值是不是 ref：

```ts
const count = ref(1)

isRef(count) // true
isRef(1) // false
```

### unRef

如果传入的是 ref，返回 `.value`；如果传入的是普通值，原样返回：

```ts
unRef(ref(1)) // 1
unRef('hello') // 'hello'
```

它常用于后续模板、代理、工具函数里做自动解包。

## 四、核心实现

当前 `ref` 内部通过 `RefImpl` 类实现：

```ts
class RefImpl<T> {
  public readonly [RefFlags.IS_REF] = true
  private _value: T
  private _rawValue: T

  constructor(value: T) {
    this._rawValue = value
    this._value = toReactive(value)
  }

  get value(): T {
    return this._value
  }

  set value(newValue: T) {
    if (Object.is(newValue, this._rawValue)) {
      return
    }

    this._rawValue = newValue
    this._value = toReactive(newValue)
  }
}
```

这里有两个内部值：

| 字段 | 作用 |
| --- | --- |
| `_rawValue` | 保存原始值，用来判断新旧值是否相同 |
| `_value` | 对外返回的值，如果是对象，会被转成 reactive |

## 五、为什么需要 _rawValue

`_rawValue` 用来避免重复设置相同值。

例如：

```ts
const count = ref(1)

count.value = 1
```

新值和旧值相同，setter 会直接 return：

```ts
if (Object.is(newValue, this._rawValue)) {
  return
}
```

使用 `Object.is` 的原因是它比 `===` 更适合做值变化判断，例如可以正确处理 `NaN`。

## 六、对象值如何处理

如果 `ref` 接收的是对象，它会自动转成 `reactive`：

```ts
const state = ref({
  count: 1
})

state.value // reactive proxy
```

实现方式：

```ts
function toReactive<T>(value: T): T {
  return isObject(value) ? reactive(value) as T : value
}
```

这样可以保证：

```ts
const state = ref({
  nested: {
    count: 1
  }
})
```

访问 `state.value` 时，第一层对象是响应式代理；访问 `state.value.nested` 时，嵌套对象也会通过 `reactive` 的懒代理逻辑变成响应式代理。

## 七、重新赋值对象

当 `.value` 被重新赋值为一个新对象时，新对象也会转成 `reactive`：

```ts
const state = ref({
  count: 1
})

state.value = {
  count: 2
}
```

setter 中会重新执行：

```ts
this._value = toReactive(newValue)
```

所以新的对象值仍然具备响应式能力。

## 八、ref 标记

当前通过内部标记判断一个对象是否是 ref：

```ts
export enum RefFlags {
  IS_REF = '__v_isRef'
}
```

`RefImpl` 实例上会带有这个标记：

```ts
public readonly [RefFlags.IS_REF] = true
```

`isRef` 通过 `Reflect.get` 读取：

```ts
export function isRef(value: unknown): value is Ref {
  return isObject(value) && Reflect.get(value, RefFlags.IS_REF) === true
}
```

这里不用普通属性访问，是为了避免把内部标记暴露成业务类型的一部分。

## 九、单元测试覆盖点

测试文件：

```text
packages/reactivity/__test__/ref.spec.ts
```

当前测试覆盖了 6 个场景。

### 1. 普通值包装

```ts
const count = ref(1)

expect(count.value).toBe(1)

count.value = 2

expect(count.value).toBe(2)
```

验证基础类型可以通过 `.value` 读写。

### 2. isRef 判断

```ts
const count = ref(1)
const state = reactive({ count: 1 })

expect(isRef(count)).toBe(true)
expect(isRef(state)).toBe(false)
expect(isRef(1)).toBe(false)
```

验证 ref、reactive 对象、普通值可以被正确区分。

### 3. unRef 解包

```ts
expect(unRef(ref(1))).toBe(1)
expect(unRef('hello')).toBe('hello')
```

验证传入 ref 时返回 `.value`，传入普通值时原样返回。

### 4. 对象值转 reactive

```ts
const original = {
  nested: {
    count: 1
  }
}
const state = ref(original)

expect(isReactive(state.value)).toBe(true)
expect(state.value).not.toBe(original)
expect(isReactive(state.value.nested)).toBe(true)
```

验证 `ref({})` 内部对象会转成响应式对象。

### 5. 重新赋值对象

```ts
const state = ref({
  count: 1
})
const next = {
  count: 2
}

state.value = next

expect(state.value).not.toBe(next)
expect(isReactive(state.value)).toBe(true)
expect(state.value.count).toBe(2)
```

验证 `.value` 设置为新对象后，新对象也会被代理。

### 6. 重复赋相同值

```ts
const state = ref({
  count: 1
})
const current = state.value

state.value = current

expect(state.value).toBe(current)
```

验证相同值不会触发无意义替换。

## 十、和 reactive 的关系

`ref` 依赖 `reactive` 来处理对象值：

```ts
import { reactive } from './reactive'
```

两者分工：

| API | 适合处理 |
| --- | --- |
| `reactive` | 对象、数组、带 symbol key 的对象 |
| `ref` | 任意值，包括基础类型和对象 |

当传入对象时：

```ts
ref({ count: 1 })
```

内部会变成：

```ts
reactive({ count: 1 })
```

当传入基础类型时：

```ts
ref(1)
```

只会被包装成：

```ts
{ value: 1 }
```

## 十一、当前还没有实现的能力

当前 `ref` 是最小版本，还没有实现依赖收集和触发更新。

后续可以继续补：

- `trackRefValue`
- `triggerRefValue`
- `shallowRef`
- `triggerRef`
- `proxyRefs`
- `toRef`
- `toRefs`
- `customRef`

这些能力通常会在 `effect`、依赖收集、调度器完善之后继续扩展。
